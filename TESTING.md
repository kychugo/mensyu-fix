| 步驟 | 操作 | 預期結果 |
|------|------|---------|

## 18. 已修復的 Bug 說明

### Bug #1 — `admin/essays.php`：closeModal() 未重置表單
**症狀：** 編輯文章後關閉模態框，再點「新增文章」，隱藏欄位仍保留 `action=edit` 和舊的 `id`。提交後會更新舊文章而非新增。  
**修復：** `closeModal()` 現在調用 `form.reset()`、重置 `action=add` 及 `id=''`。  
**測試：** ESS-07、ESS-08

### Bug #2 — `admin/matching_data.php`：closeModal() 未重置表單
**症狀：** 與 Bug #1 相同，影響配對題組管理。  
**修復：** 同上，`closeModal()` 重置表單及隱藏欄位。  
**測試：** PAIR-08、PAIR-09

### Bug #3 — `games/matching.php`：難度過濾不生效
**症狀：** `startGame()` 中過濾條件為 `return true`，所有難度均從同一完整詞庫隨機抽取，選難度僅影響題目數量和時間限制。SQL 查詢也未返回 `difficulty` 欄位。  
**修復：** SQL 查詢加入 `difficulty` 欄位；JS 先按選定難度過濾，不足時再補充全部詞組。  
**測試：** GAME-02、GAME-03、GAME-04

### Bug #4 — `learning/quiz.php`：選項含單引號導致 JS 錯誤
**症狀：** `onAnswer(${i}, '${escHtml(opt)}')` 在選項含 `'` 時，`&#39;` 被瀏覽器 HTML 解碼還原為 `'`，切斷 JS 字串，造成 `SyntaxError`，選項高亮完全失效。  
**修復：** 移除 `onAnswer()` 的第二個參數（該參數原本就未被使用），改為 `onAnswer(${i})`。  
**測試：** QUIZ-03、QUIZ-04

### Bug #5 — `admin/matching_data.php` 批量匯入：XLSX 標題行未過濾
**症狀：** 純文字/CSV 匯入會跳過「文言字詞, classical, 字詞, term」等標題行。但 XLSX 的解析路徑沒有應用同樣的過濾邏輯，上傳有標題行的 XLSX 時，標題行（如「文言字詞 → 現代語譯」）會被當成真實題組匯入。  
**修復：** 在 XLSX 的 `.filter()` 中加入與 `parseBulkText` 相同的標題行正則過濾。  
**測試：** BULK-12

### Bug #6 — `admin/matching_data.php` 批量匯入：無資料庫事務
**症狀：** 批量 INSERT 逐行執行，無事務保護。若在第 N 行發生資料庫錯誤，前 N-1 行已永久插入，資料不一致。  
**修復：** 批量匯入整個循環包裝在 `$db->beginTransaction()` / `$db->commit()` 中；異常時執行 `$db->rollBack()` 並顯示錯誤訊息。  
**測試：** BULK-08（觸發部分行有效/部分行無效的場景）

### Bug #7 — `learning/quiz.php`：XP 每次通關都發放
**症狀：** 測驗每次通過均調用 `addUserXp()`，重複通關同一關卡可無限刷 XP。設計意圖應為只在首次通關（或舊記錄尚未通關）時發放。  
**修復：** 在發放 XP 前檢查 `$row['completed']`，若舊記錄已是「已完成」狀態則不再重複發放。  
**測試：** QUIZ-14、QUIZ-15

### Bug #8 — `social/index.php`：留言驗證失敗靜默重定向
**症狀：** 提交空白留言或超過 200 字的留言時，原來的代碼直接 `header('Location: ...)` 跳轉，不顯示任何錯誤訊息，用戶不知道留言為何沒有發出。  
**修復：** 驗證失敗時設置 `$postError`，不執行跳轉，讓頁面顯示錯誤提示（空留言、超出字數上限、無效 post_id 均有對應訊息）。  
**測試：** SOC-03（空留言）、SOC-04（超出字數）
